Tipos de Variáveis em JavaScript

JavaScript é uma linguagem de programação dinâmica que suporta vários tipos de variáveis. Essas variáveis são usadas para armazenar diferentes tipos de dados, como números, texto e objetos. Alguns dos principais tipos de variáveis em JavaScript incluem:

  • Números (Number): Usados para representar valores numéricos, como inteiros e números de ponto flutuante.
  • Strings: Usados para representar texto. As strings são definidas entre aspas simples ou duplas.
  • Booleanos (Boolean): Representam valores verdadeiro ou falso, úteis para lógica condicional.
  • Objetos (Object): São estruturas de dados que podem conter múltiplos valores e métodos.
  • Arrays: São objetos especiais usados para armazenar coleções ordenadas de valores.
  • Null e Undefined: São tipos que representam valores nulos ou indefinidos. 
let idade = 30; // Variável 'idade' armazena um número

Importação de Arquivos JavaScript

Para utilizar JavaScript em uma página HTML, você precisa importar um arquivo .js com a tag <script> no cabeçalho (head) ou no corpo (body) do documento. Isso permite que você execute código JavaScript na página.

Existem duas maneiras principais de importar um arquivo JavaScript:

1. Importação no Cabeçalho:

Você pode adicionar a tag <script> diretamente no cabeçalho do documento. Isso carregará o script antes de renderizar o conteúdo da página. No entanto, se você não usar o atributo defer ou async, o navegador pode bloquear a renderização da página até que o script seja totalmente carregado e executado.

Exemplo:

<script src="seu-arquivo.js"></script>

2. Importação no Corpo com Defer:

Uma prática recomendada é adicionar a tag <script> no corpo do documento, geralmente antes do fechamento da tag </body>, e usar o atributo defer. Isso permite que o navegador continue a renderizar a página enquanto o script é carregado em segundo plano. O script é executado apenas após a renderização completa da página.

Exemplo com defer:

<script src="seu-arquivo.js" defer></script>

A opção defer é útil quando você deseja garantir que seu código JavaScript não bloqueie a renderização da página e seja executado na ordem correta após o carregamento completo da página. É especialmente importante quando você possui scripts que dependem da estrutura HTML da página.

Lembre-se de que você também pode usar o atributo async se a ordem de execução não for crítica e você desejar que o script seja carregado e executado assim que possível, independentemente da ordem em que aparece na página.

Manipulação do DOM (Document Object Model)

O DOM representa a estrutura da página web como uma árvore de objetos. JavaScript pode ser usado para acessar e manipular elementos HTML através do DOM:

Exemplo de alteração de conteúdo de um elemento:

<div id="meuElemento">Texto antigo</div>
let elemento = document.getElementById("meuElemento");
            elemento.innerHTML = "Novo conteúdo";

Exemplo de adição de um novo elemento:

<div id="container"></div>
let container = document.getElementById("container");
            let novoElemento = document.createElement("p");
            novoElemento.textContent = "Novo parágrafo";
            container.appendChild(novoElemento);

Exemplo de remoção de um elemento:

<div id="elementoParaRemover"></div>
let elementoParaRemover = document.getElementById("elementoParaRemover");
            elementoParaRemover.parentNode.removeChild(elementoParaRemover);

Variáveis e Tipos em JavaScript

Em JavaScript, você pode declarar variáveis usando let, const e var.

Essas diferenças existem para fornecer controle sobre o escopo e a mutabilidade das variáveis, permitindo que você escolha a ferramenta certa para o trabalho.


let

  • Tem escopo de bloco, ou seja, é visível apenas dentro do bloco onde foi declarado.
  • Pode ser reatribuído (ter seu valor alterado).
  • Recomendado para a maioria das variáveis devido ao escopo mais restrito.

const

  • Também tem escopo de bloco.
  • Não pode ser reatribuído (seu valor não pode ser alterado).
  • Recomendado para variáveis cujo valor não deve mudar após a atribuição inicial.

var

  • Tem escopo de função ou escopo global.
  • Pode ser reatribuído.
  • Menos recomendado devido ao escopo mais amplo e potencial para comportamento inesperado.
